BEGINNER FIRST

まず、言葉とイメージをつかむ

たとえると

教室の案内表示は、情報が正しくても文字が小さい、色だけで区別する、場所が毎回変わると使いにくくなります。UIは見た目だけでなく、目的を達成できる操作設計です。

UI

ボタン、文字、グラフなど利用者が接する画面や操作部分。

UX

利用前から利用後までを含む利用体験。

フィードバック

操作結果を画面・音・振動などで返すこと。

一貫性

同じ意味の操作や表示を同じ方法で表すこと。

アクセシビリティ

多様な利用者が情報や機能へアクセスできるようにすること。

CONCEPT GUIDE

見た目だけでなく、目的を達成できるか確かめる

最初は要点だけ確認し、必要な説明を開いて学びます。理解に必須の図は常に表示し、詳しい読み方・誤解しやすい点を開閉式に整理しています。

使う場面

利用者が迷わず、誤操作を減らし、必要な情報へ到達できる画面を設計するとき。

注意する場面

色や装飾だけで評価するとき。読み込み中・空状態・エラー・キーボード操作も設計します。

改善前と改善後を比べる

単位、更新時刻、操作後のフィードバックを示し、色だけに依存しない表示へします。

通常時以外の状態を設計する

完了完了時刻、受付番号、次の行動
アクセシビリティの最小確認
  • 入力欄に見えるラベルを付ける
  • キーボードだけで操作できる
  • フォーカス位置が見える
  • 色だけで状態を区別しない
  • エラー内容を文章で示す
  • 文字と背景のコントラストを確認する

CODE / COLAB

最小コードを動かす

REVIEWED MINIMUM EXAMPLE
<!-- 最小の操作可能プロトタイプ -->
<button id="refresh" type="button">最新値を表示</button>
<p id="status" aria-live="polite">未更新</p>
<script>
  document.getElementById("refresh").addEventListener("click", () => {
    document.getElementById("status").textContent = "更新しました 10:30";
  });
</script>

コードは学習用の最小例です。ファイル名、列名、単位、欠損、ライブラリのバージョンを確認して実行してください。

CODE WALKTHROUGH

このコードは、どう動く？

ボタンがクリックされたときにJavaScriptが動き、画面内の文字を更新する最小プロトタイプです。 コード全体を一度に読むのではなく、入力がどのように変化して出力になるかを順番に追います。

各ステップの詳しい解説

DATA / DESIGN MISSION

センサダッシュボードのUIを試作する

値・単位・測定場所・更新時刻・注意事項を表示し、色だけに依存しない状態表現を作ります。キーボード操作と文字の読みやすさも確認します。
