PROCESS

  1. 1対象者と問題を40秒以内で示す
  2. 2データの出典・方法・主要結果を絞る
  3. 3プロトタイプの仕組みとデモを見せる
  4. 4検証結果と失敗・限界を示す
  5. 5次の改善と協力してほしいことを伝える

CODE / COLAB

最小コードを動かす

REVIEWED MINIMUM EXAMPLE
# 5分ピッチの時間配分を確認する簡単な例
sections = {
    "問題": 40,
    "データと方法": 60,
    "結果": 60,
    "プロトタイプとデモ": 90,
    "検証・限界・次の改善": 50
}
print("合計秒:", sum(sections.values()))
for name, sec in sections.items():
    print(name, f"{sec//60}:{sec%60:02d}")

コードは学習用の最小例です。ファイル名、列名、単位、欠損、ライブラリのバージョンを確認して実行してください。

CODE WALKTHROUGH

このコードは、どう動く？

5分間の発表構成を秒数で定義し、合計時間と各パートの分・秒表示を確認するコードです。 コード全体を一度に読むのではなく、入力がどのように変化して出力になるかを順番に追います。

各ステップの詳しい解説

DATA / DESIGN MISSION

5分発表＋デモを完成させる

問題、データ、分析、プロトタイプ、検証、限界をつなげます。分析手法の説明会にならないよう、手法は「問いに答えるためにどう使ったか」を中心にします。

解釈と安全の注意

グラフの出典・単位・年次を省略しません。小さな結果を誇張せず、失敗・限界・今後の検証も成果として伝えます。

CBT / KNOWLEDGE + PERFORMANCE

基礎理解＋操作判断テスト

用語だけでなく、計算・データ処理・判断を含む4問です。採点後に解説が表示されます。
