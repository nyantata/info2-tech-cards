IoT全体構成図

センサ値はそのまま意味になるとは限りません。校正、単位変換、欠損処理、条件判定を経て利用します。

測定間隔と見落とし

短い間隔

速い変化を捉えやすい一方、データ量・通信量・電力消費が増えます。

長い間隔

データ量は減りますが、瞬間的な変化やピークを見落とす可能性があります。

判断例：「10秒間隔で記録したため、10秒より短い変化は十分に捉えられない」と限界をログへ残します。
ノイズ・欠損・移動平均

移動平均は短い揺れをならしますが、急な変化を遅らせることがあります。元データを消さず、平滑化した列を別に作るのが基本です。

CODE / COLAB

最小コードを動かす

REVIEWED MINIMUM EXAMPLE
import pandas as pd

df = pd.read_csv("assets/data/sensor_demo_simulated.csv")
df["timestamp"] = pd.to_datetime(df["timestamp"])

# 長い形式のCSVから照度だけを取り出す
light = (
    df[df["sensor_name"] == "light"]
    .sort_values("timestamp")
    .copy()
)

# 5件の移動平均で短い揺れをならす
light["value_ma5"] = light["value"].rolling(5).mean()
print(light[["timestamp", "value", "unit", "value_ma5"]].head(10))

コードは学習用の最小例です。ファイル名、列名、単位、欠損、ライブラリのバージョンを確認して実行してください。

CODE WALKTHROUGH

このコードは、どう動く？

センサログを読み込み、時刻として扱える形に変換し、移動平均で短い揺れをならすコードです。 コード全体を一度に読むのではなく、入力がどのように変化して出力になるかを順番に追います。

解釈と安全の注意

照度、温度、振動などの値だけから、心理状態・健康状態・能力を直接断定しません。センサ値は設置位置、個体差、測定間隔の影響を受けます。
