SETUP GUIDE

Google連携の設定

学校のGoogleアカウントでログインし、提出内容を先生のスプレッドシートへ保存するための設定です。

1．記録先のスプレッドシートを作る

先生のGoogleドライブに新しいスプレッドシートを作成し、URL中のスプレッドシートIDを控えます。シート名や見出し行はスクリプトが自動作成します。

2．Apps Scriptを作成する

Apps Scriptの新しいプロジェクトを作り、同梱した gas/Code.gs と gas/appsscript.json の内容を配置します。

スクリプトプロパティ

名前設定値
SPREADSHEET_ID手順1で控えたID
GOOGLE_CLIENT_ID手順4で作るウェブ用クライアントID
ALLOWED_DOMAIN_SUFFIX.ed.jp または学校の正確なドメイン
SHEET_NAME提出ログ（任意）

3．ウェブアプリとして公開する

Apps Scriptの「デプロイ」から「新しいデプロイ」→「ウェブアプリ」を選びます。実行するユーザーは先生、アクセスできるユーザーは学校の運用方針に合わせて設定します。外部の静的サイトから利用する場合でも、スクリプト側でGoogleのIDトークンとed.jpドメインを検証します。

公開後のウェブアプリURLを控えます。

4．Googleログイン用のクライアントIDを作る

Google CloudでOAuth 2.0の「ウェブ アプリケーション」クライアントを作成し、技術カードサイトの公開元を「承認済みのJavaScript生成元」に登録します。学校内だけで使う場合は、OAuth同意画面を組織内向けに設定します。

5．サイト側の設定を入力する

assets/gas-config.jsを開き、ウェブアプリURLとクライアントIDを入力します。

window.INFO2_GAS_CONFIG = {
  webAppUrl: 'Apps ScriptのウェブアプリURL',
  googleClientId: 'xxxxx.apps.googleusercontent.com',
  allowedDomainSuffix: '.ed.jp'
};

6．動作確認

  1. 学校のGoogleアカウントでログインする
  2. テスト用ログを入力し、確認画面を開く
  3. 「提出」を押してスプレッドシートへ記録されることを確認する
  4. 「自分のログ」で本人の提出だけが表示されることを確認する
