BEGINNER FIRST

まず、言葉とイメージをつかむ

たとえると

複数の教科の得点に共通して関係する「読解力」「計算力」のような要因があると考えるイメージです。ただし、その要因を直接測ったわけではなく、データの関係から推定します。

観測変数

アンケート項目など、実際に測定された変数。

共通因子

直接は測れないものの、複数の質問や測定値に共通して影響していると考える要因。

因子負荷量

各質問と因子が、どのくらい強く結びついているかを表す数値。プラスとマイナスは関係の向きを表します。

独自性

その質問だけに特有で、共通因子では説明できない部分。

回転

因子の意味を読み取りやすくするため、因子の軸の向きを変える処理。

CONCEPT GUIDE

観測項目の背後にある共通したまとまりを考える

まず図と要点を確認します。さらに知りたい内容は、見出しをクリックすると下に開きます。

使う場面

多くの質問項目を、回答の似た動きに注目して、少数の共通するまとまりへ整理するとき。

気をつけること

因子の名前や個数には、1つだけの正解があるわけではありません。質問内容、回答者数、因子の個数、回転方法によって結果が変わります。

観測項目と、見えない共通要因

因子は直接測った値ではありません。複数の質問への回答が似た動きをする理由を説明するために考えた、見えない共通要因です。

回転・符号・順序の注意

回転は、因子の意味を読み取りやすくするための変換です。因子1と因子2の順番が入れ替わることがあります。また、すべてのプラスとマイナスが逆になっても、表している関係は同じ場合があります。

PROCESS

手順に分けて進める

  1. 1目的に沿った複数項目を準備する
  2. 2未回答の値、回答の段階、質問どうしの相関を確認する
  3. 3因子を何個にするか、どの方法で求めるかを検討する
  4. 4因子負荷量と独自性を確認する
  5. 5因子名を付け、その理由と別の読み方の可能性を書く

HANDS-ON

  1. STEP 1

  3. STEP 3

CODE / COLAB

最小コードを動かす

REVIEWED MINIMUM EXAMPLE
import pandas as pd
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import FactorAnalysis

df = pd.read_csv("assets/data/factor_demo_simulated.csv")
items = ["q1", "q2", "q3", "q4", "q5", "q6"]
X = df[items].dropna()
Z = StandardScaler().fit_transform(X)

model = FactorAnalysis(n_components=2, rotation="varimax", random_state=42)
model.fit(Z)
loadings = pd.DataFrame(
    model.components_.T, index=items, columns=["Factor1", "Factor2"]
)
print(loadings.round(3))

コードは学習用の最小例です。ファイル名、列名、単位、欠損、ライブラリのバージョンを確認して実行してください。

解釈と安全の注意

因子分析は、因子を客観的に「発見して名前まで自動決定する」方法ではありません。因子の個数、回転方法、質問項目、回答者の集まりによって結果が変わります。
