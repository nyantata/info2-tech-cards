使う場面

気をつけること

特徴量の単位や数値の大きさをそろえないと、距離を公平に比べられません。kを変えた理由も説明します。使う特徴量が多すぎると、近い・遠いの差が分かりにくくなる場合があります。

近傍点と投票の内訳

どの点が選ばれたか、各点までの距離、A/Bの票数を示すと予測理由を説明できます。

kが小さい／大きい場合

小さいk

近くにある少数の点の影響を強く受けるため、測定の揺れや大きく離れた点に左右されやすくなります。

大きいk

判定の境界はなめらかになりますが、少数だけ存在する特徴的なグループを見落とすことがあります。

PROCESS

手順に分けて進める

  1. 1特徴量と正解の分類名（ラベル）を準備する
  2. 2訓練データとテストデータへ分ける
  3. 3訓練データの平均と標準偏差だけを使って、標準化の基準を決める
  4. 4kを変え、データの分け方を変えて何度か評価する「交差検証」で比べる
  5. 5テストデータで正解率と混同行列を確認する

CODE / COLAB

最小コードを動かす

REVIEWED MINIMUM EXAMPLE
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.neighbors import KNeighborsClassifier
from sklearn.metrics import accuracy_score, confusion_matrix

X, y = load_iris(return_X_y=True)
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, random_state=42, stratify=y
)
model = make_pipeline(StandardScaler(), KNeighborsClassifier(n_neighbors=5))
model.fit(X_train, y_train)
pred = model.predict(X_test)
print("テストデータ正解率:", accuracy_score(y_test, pred))
print(confusion_matrix(y_test, pred))

コードは学習用の最小例です。ファイル名、列名、単位、欠損、ライブラリのバージョンを確認して実行してください。

解釈と安全の注意

距離にもとづくため、変数ごとの数値の範囲の影響を受けます。モデル作成に使っていないテストデータを、標準化の平均や標準偏差の計算へ混ぜないようにします。scikit-learnのPipelineで、標準化と分類を一つの手順にまとめると、正しい順序で処理しやすくなります。
