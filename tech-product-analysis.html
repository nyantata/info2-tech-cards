BEGINNER FIRST

まず、言葉とイメージをつかむ

たとえると

自動販売機を分解して考えると、利用者がボタンを押すことが入力、在庫と金額の確認が処理、商品と表示が出力です。補充や故障対応は運用に当たります。

利用者

システムを使う人や組織。

入力

センサ値、文字、画像、操作など、システムに入る情報。

処理

入力を計算・分類・検索・変換すること。

出力

画面、音、LED、通知など、処理結果を利用者へ返すもの。

運用

安全性、保守、権限、障害対応まで含めて使い続けること。

CODE / COLAB

最小コードを動かす

REVIEWED MINIMUM EXAMPLE
def classroom_monitor(light_lx, temperature_c):
    # 教材用の単純なルール。実運用の基準ではない。
    if light_lx < 250:
        return "照明条件を確認"
    if temperature_c > 30:
        return "空調条件を確認"
    return "記録を継続"

print(classroom_monitor(220, 27))

コードは学習用の最小例です。ファイル名、列名、単位、欠損、ライブラリのバージョンを確認して実行してください。

CODE WALKTHROUGH

このコードは、どう動く？

入力された照度と温度を、上から順番に条件判定してメッセージを返すコードです。 コード全体を一度に読むのではなく、入力がどのように変化して出力になるかを順番に追います。

DATA / DESIGN MISSION

身近な情報システムを構成図にする

学校の出欠、図書館、交通案内などから1つ選び、公開情報と実際の操作をもとに構成図を作ります。内部仕様が確認できない部分は「推測」と明記します。

確認先とデータ方針

解釈と安全の注意

外から見える動作だけで内部のアルゴリズムを断定しないこと。個人情報や認証情報を収集・掲載しないこと。
