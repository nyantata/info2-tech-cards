使う場面

条件分岐をたどって説明できる、分類や数値予測のモデルを作るとき。

気をつけること

分岐が多すぎる木は、モデル作成に使ったデータへ合わせすぎることがあります。また、分岐条件が原因や普遍的なルールを表しているとは限りません。

根から葉まで経路をたどる

実際のIrisデータではクラス0=setosa、1=versicolor、2=virginicaです。学習した木の条件は訓練データと設定で変わります。

分岐は「混ざり具合」を減らす

分岐前

複数クラスが混ざっている。

分岐後

子ノードでクラスがより分かれ、混ざり具合が小さくなる条件を選ぶ。

ジニ不純度は、クラスの混ざり具合を表す数値です。0なら、その場所のデータは1種類のクラスだけです。

深さと過学習

深さを増やすと訓練データ正解率は上がりやすい一方、テストデータ正解率が下がることがあります。木の深さを決めるmax_depthや、葉に必要な件数を決めるmin_samples_leafを変え、同じ条件や交差検証で比べます。

CODE / COLAB

最小コードを動かす

REVIEWED MINIMUM EXAMPLE
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split
from sklearn.tree import DecisionTreeClassifier, export_text
from sklearn.metrics import accuracy_score

X, y = load_iris(return_X_y=True)
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, random_state=42, stratify=y
)
model = DecisionTreeClassifier(max_depth=3, random_state=42)
model.fit(X_train, y_train)
pred = model.predict(X_test)
print("テストデータ正解率:", accuracy_score(y_test, pred))
print(export_text(model, feature_names=list(load_iris().feature_names)))

コードは学習用の最小例です。ファイル名、列名、単位、欠損、ライブラリのバージョンを確認して実行してください。

DATA / DESIGN MISSION

Irisデータで深さの異なる決定木を比較する

同じ訓練・テスト分割でmax_depthだけを変え、訓練データ正解率とテストデータ正解率を比較します。木の図だけでなく誤分類も確認します。
